# Security

Security is central to the Replant vision.

Because Replant may involve sensitive prayer requests, church communication, resource needs, and persecuted church concerns, the platform must be designed with privacy, safety, and accountability in mind.

## Early Security Considerations

Future development should consider:

- Secure authentication
- Church or organization verification
- Role-based access controls
- Encrypted communication
- Abuse reporting
- Internal safety review workflows
- Logging and audit trails for moderation actions
- Clear privacy policy and user expectations

## Moderation Philosophy

Replant should protect communication from being exposed publicly while still maintaining safety inside the platform.

Users should understand that certain harmful, suspicious, or unsafe language may trigger internal review.

## Current Status

No production system exists yet. This document captures early security expectations for future planning.
