# Technical Vision

Replant will likely require both web and mobile access over time.

## Possible Architecture Areas

- Frontend web application
- Backend API
- Authentication service
- Database
- Messaging or posting system
- Moderation and review system
- Notification system
- Admin dashboard

## Important Technical Needs

- Secure login
- Verified organization access
- Role-based permissions
- Encrypted data in transit
- Careful handling of sensitive content
- Audit logs for admin and moderation actions
- Scalable design for regional growth

## Open Questions

- Should Replant begin as web-only before mobile?
- What user roles are required for MVP?
- What content should be encrypted end-to-end?
- What content needs moderation visibility?
- How should churches be verified?
