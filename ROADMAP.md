# Roadmap

## Phase 1: Foundation

- Clarify vision
- Define core principles
- Document technical direction
- Gather church leader feedback
- Launch informational website

## Phase 2: Product Planning

- Define user roles
- Map church onboarding flow
- Define prayer request and needs-sharing structure
- Define persecuted church section
- Draft privacy and moderation expectations

## Phase 3: Prototype

- Build clickable wireframes
- Validate with a small group of churches
- Refine core workflows
- Identify technical architecture

## Phase 4: MVP

- Build initial web experience
- Implement authentication
- Add church profiles or groups
- Add prayer request posting
- Add moderation/reporting workflow
- Test with a small pilot group

## Phase 5: Expansion

- Improve security
- Add mobile support
- Expand regional church connections
- Add resource-sharing features
- Prepare for broader onboarding
